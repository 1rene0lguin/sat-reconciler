Más que código: Decisiones de Arquitectura

Esta herramienta es el resultado de 16 años resolviendo problemas fiscales de alto volumen. Así es como transformamos lecciones de la industria en software de precisión.

El Desafío de la Escala Real

Lección aprendida: Gestionando la nómina de un Gigante Global de Cosméticos (Fortune 500), descubrimos que la API estándar del SAT fallaba bajo concurrencia extrema.

> Reingeniería Aplicada:
Este motor aplica los algoritmos de rate-limiting y particionado que diseñé para resolver esos cuellos de botella masivos, reescritos desde cero en Go.

Minería de Datos & Compresión

El Problema: Almacenar millones de XMLs en FileSystem es ineficiente y lento para auditoría.

> Arquitectura:
Almacenamiento NoSQL/Embedded con compresión de bits en atributo.
Resultado: 50% menos storage vs ZIPs tradicionales y acceso instantáneo para Data Mining.

Seguridad "On-Premise"

Riesgo: Subir la FIEL a la nube de terceros es un riesgo de seguridad inaceptable para corporativos.

> Modelo de Responsabilidad:
El backend procesa en memoria (RAM) y desecha las llaves inmediatamente.
Próximamente: Versión Desktop aislada (Air-gapped ready).

Por qué Go (Golang)

Performance: Node.js y Python consumen demasiada memoria procesando XMLs masivos.

> Benchmarks:
Compilación nativa estática.
Manejo de miles de Goroutines para descarga concurrente sin bloquear el hilo principal.

📄 Flujo de Solicitud

  1. 1. Datos y Rango

    Se define el rango de fechas y tipo de solicitud (Metadata).

  2. 2. Firma Digital (XMLDSig)

    El backend usa tu FIEL para firmar el sobre SOAP. Tus llaves solo existen en RAM.

  3. 3. Recepción de UUID

    El SAT devuelve un ID de seguimiento. Guárdalo para el Paso 2.

Nota Importante: Este paso NO descarga los archivos inmediatamente. El SAT requiere un tiempo de procesamiento (minutos a horas).

🔎 Flujo de Verificación

  1. 1. Consulta de Estado

    Se pregunta al SAT si el UUID ya está "Terminado".

  2. 2. Autorización de Descarga

    Si hay paquetes, se habilita el Paso 3.

  3. 3. Re-Autenticación

    Por seguridad, debes subir tu FIEL nuevamente para descargar.
